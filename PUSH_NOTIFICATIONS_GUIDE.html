🚀 Quick Start (3 passos)

1️⃣ Gerar VAPID Keys

npm install -g web-push
web-push generate-vapid-keys

Guardar as keys num local seguro!

2️⃣ Configurar Vercel

  1. Dashboard Vercel → Settings → Environment Variables
  2. Adicionar: 
    NEXT_PUBLIC_VAPID_PUBLIC_KEY=BJOU...
VAPID_PRIVATE_KEY=R4nC...
SUPABASE_URL=https://...
SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
  3. Deploy

3️⃣ Executar SQL no Supabase

  1. Supabase → SQL Editor
  2. Copiar conteúdo de PUSH_NOTIFICATIONS.sql
  3. Executar

🧪 Testar Localmente

// 1. No browser console
PushNotifications.getStatus()

// 2. Subscrever utilizador
await PushNotifications.subscribeUser()

// 3. Enviar teste
await PushNotifications.sendTestNotification()

// 4. Ver subscription no Supabase
SELECT * FROM push_subscriptions;

📊 Fluxo de Dados

Passo Componente Ação
1 Browser (push-notifications.js) User clica "Ativar Notificações"
2 Service Worker (sw.js) Registar + pedir permissão
3 API /api/subscribe POST subscription para Supabase
4 Supabase (push_subscriptions) Guardar subscription endpoint + keys
5 Admin (/dashboard-push.html) Entra título/mensagem e clica enviar
6 API /api/send-notification Lê subscriptions do Supabase
7 web-push (web-push package) Envia via VAPID para FCM/APNs
8 Service Worker (push event) Recebe e exibe notificação
9 Browser/Device Utilizzador vê notificação! 🎉

🔐 Segurança — Checklist

  • VAPID Private Key guardada apenas em Vercel (server-side)
  • VAPID Public Key exposta via /api/vapid-key (seguro)
  • RLS policies: INSERT público, SELECT bloqueado
  • HTTPS automático (Vercel)
  • Validação de input (title, body required)
  • Auto-cleanup de subscrições inválidas (410, 404)

⚙️ Environment Variables (Vercel)

Variável Origem Tipo Uso
NEXT_PUBLIC_VAPID_PUBLIC_KEY web-push generate-vapid-keys Público Frontend (crypto)
VAPID_PRIVATE_KEY web-push generate-vapid-keys Confidencial Backend (sign)
SUPABASE_URL Supabase → Settings → API Público Cliente + Servidor
SUPABASE_ANON_KEY Supabase → Settings → API Público Cliente (INSERT subscriptions)
SUPABASE_SERVICE_ROLE_KEY Supabase → Settings → API Confidencial Servidor (SELECT, DELETE subscriptions)
⚠️ IMPORTANTE: Nunca comitar environment variables em Git! Use .env.local (gitignore) para desenvolvimento local.

🐛 Troubleshooting Rápido

Problema Causa Solução
Service Worker não registado Ficheiro não encontrado / HTTP em prod Verificar /sw.js existe + HTTPS
"Permission denied" User recusou no dialog Ir a Settings → Notifications e permitir
Notificação não aparece Browser em background ou erros DevTools → Application → Service Workers
API returning 410/404 Subscription expirada Removida automaticamente! Check Supabase
CORS error Requesting dari HTTP Usar /api/* routes (same-origin)
